To allow Brenda-Web access to create and access your AWS account and its resources, it needs to be given credentials that are linked to your account.

  1. If you haven't already done so, create an AWS account
  2. Log in to the AWS web interface and click on "Identity & Access Management" found under "Security & Identity"
  3. Click on the Users link which will take you to a list of the current users configured for this account. For fresh accounts, there should be a single user. For the sake of simplicity, we'll leverage the existing user. If you so desire, you may create a new user or isolated linked accounts but doing so is beyond the scope of this tutorial. For more information on security credentials, see the documentation here.
  4. Click on the user you'd like to use, then click on the Security Credentials tab.
  5. You should now see a Create Access Key button, click on it.
  6. A dialog will open with your security credentials. You may display them in the dialog or optionally download them to a CSV file. Note that once the dialog is closed, the credentials will no longer be accessible so make sure to grab them and place them somewhere secure.
  7. At the top of the Brenda-Web page, click on the AWS Config button. This will open a dialog in which the AWS credentials should be inserted. Place the access key ID and secret into their corresponding boxes in the dialog.
  8. For this tutorial, enter the value us-east-1 in the AWS Region text box. For more information on what region means, refer to the AWS Regions and Availability Zones documentation.
  9. Once the access key ID, secret, and region have been populated, click the Set Credentials button

Clicking Set Credentials will cause the tool to test the provided credentials agains AWS. If this is your first time using the tool (or first time within the provided region), the security group check is likely to fail. Clicking the Fix it link that appears will let the tool attempt to create the security group in AWS.

To see the details of the created security group, it can be found by going to AWS then EC2 --> Security Groups. Here, you should see a security group named "brenda-web". This is the security group that was created and you can see the details of what was created.
